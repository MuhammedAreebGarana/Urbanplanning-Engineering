🟦 Core Server

Node.js

RESTful API Layer

Handles JSON-based data exchange between AI agents and the frontend, serving as the central orchestration hub for all service communication.

  • RESTful API endpoints for agent–frontend data exchange
  • JSON request/response serialization & validation
  • Async I/O for concurrent agent task handling
  • Route middleware for auth & rate limiting
  • IPC bridge to Docker execution sandbox
🟩 Database

PostgreSQL

+ PostGIS Extension

Manages heterogeneous geospatial and census datasets with full spatial query support, enabling complex GIS operations at the database layer.

  • PostgreSQL or MySQL for relational data
  • PostGIS extension for geospatial operations
  • Spatial indexing for high-performance GIS queries
  • Census & heterogeneous dataset management
  • Geometry types: points, polygons, rasters
🟧 Execution Sandbox

Docker CE

Linux Execution Environment

Provides an isolated, secure container for running Python simulation scripts and terminal commands, preventing host system exposure from untrusted code.

  • Dockerized Linux (Community Edition)
  • Isolated Python simulation script execution
  • Secure terminal command sandboxing
  • Resource limits: CPU, memory, network I/O
  • Ephemeral containers per execution job

Request / Execution Flow
01

Agent or Frontend Request

AI agent or UI client sends a JSON payload via HTTP to the Node.js REST API endpoint.

02

API Routing & Validation

Node.js parses, validates, and routes the request — dispatching either a database query or a sandbox execution job.

03

Geospatial / Census Query

PostgreSQL + PostGIS executes spatial queries against heterogeneous datasets, returning structured geospatial results.

04

Simulation Execution

Docker CE spawns an ephemeral Linux container, runs the Python simulation script securely, captures stdout/stderr, and returns results to Node.js.

05

JSON Response Delivered

Node.js aggregates outputs and returns a structured JSON response to the requesting AI agent or frontend client.
